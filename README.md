# Contest Platform
![CI](https://github.com/MaksimKosarevHSE/Contest-Platform/actions/workflows/ci.yml/badge.svg)

**Важно:** проект находится в активной разработке, в процессе будет добавление функционала, исправления.


Этот проект — это вертикально и горизонтально (для некоторых сервисов) мастштабируемая микросервисная система для проведения контестов или решения задач по программировнию. Она позволяет проводить контесты, управлять задачами, принимать и тестировать решения участников.

## Архитектура

Работа микросервисов подробно описана в MIRO

[Ссылка на диаграмму микросервисов в MIRO](https://miro.com/app/board/uXjVGy4zEVE=/?share_link_id=581367335863)

Основные микросервисы / компоненты:

- **API Gateway** — маршрутизация запросов, балансировщик нагрузки (пока не реализовано), аутентификация
- **Auth Service** — отвечает за аутентификацию
- **Problem Service** — хранит задачи, контесты и прогресс участников
- **Submission Service** — обрабатывает посылки участников, управляет процессом тестирования и публикует результаты.
- **Judging Service** — тестирует посылки участников

Все сервисы используют **PostgreSQL** в качестве основной базы данных, **Redis** для кэширования (таблицы лидеров, прогресс по задачам, refresh токены) и **Kafka** для асинхронного общения микросервисов. Миграции схемы БД управляются через **Liquibase**.

## Технологический стек

- **Spring Boot** — основа микросервисов
- **Spring Cloud** — управление распределенной системой
- **Spring Data JPA / Hibernate** — ORM, общение с БД
- **Liquibase** — миграции БД
- **Apache Kafka** — брокер сообщений
- **Redis** — кэш, подсчет таблиц лидеров, хранилище токенов
- **PostgreSQL** — реляционная база данных
- **Spring Messaging / WebSocket** — уведомления
